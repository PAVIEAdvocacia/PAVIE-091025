# 12.03 — Design System Institucional PAVIE para Web

## 1. Finalidade

Este documento fixa o design system mínimo executável da PAVIE para implementação do site e do blog, com ênfase em:

- sobriedade;
- legibilidade;
- consistência entre superfícies;
- autoridade institucional sem teatralidade;
- alta manutenção com baixa entropia visual.

Este documento não pretende congelar toda estética futura da marca. Pretende fornecer base suficientemente firme para que Astro, componentes, templates e CMS sejam implementados com unidade.

---

## 2. Princípios de interface

A interface da PAVIE deve transmitir:

1. clareza antes de ornamentação;
2. densidade antes de ruído;
3. conforto de leitura antes de impacto visual;
4. autoridade antes de chamatividade;
5. continuidade sistêmica antes de improviso.

### 2.1 O que isso significa na prática

Preferir:

- fundo claro e estável;
- contraste elevado;
- tipografia madura;
- grid limpo;
- respiro real;
- poucos acentos visuais;
- componentes repetíveis.

Evitar:

- excesso de badges;
- blocos chamativos sem função;
- múltiplos estilos de card;
- paleta fragmentada;
- animações decorativas pesadas;
- hero visual superlotado.

---

## 3. Paleta semântica recomendada

A paleta-base deve partir da identidade já materializada no logotipo atual: fundo quente neutro, tinta escura e acento vinho.

## 3.1 Tokens-base

| Token | Valor sugerido | Uso |
|---|---:|---|
| `--color-ink` | `#111111` | texto principal, ícones, linhas dominantes |
| `--color-ink-soft` | `#2D2A27` | texto secundário forte |
| `--color-surface-0` | `#F6F1ED` | fundo geral |
| `--color-surface-1` | `#EDE4DC` | cards, caixas, destaques suaves |
| `--color-brand-warm` | `#C9BEB5` | superfícies de apoio e assinatura cromática |
| `--color-accent` | `#971D25` | detalhe institucional, links ativos, ênfase contida |
| `--color-accent-strong` | `#7E171C` | hover, foco e reforço de destaque |
| `--color-line` | `#D7CCC3` | bordas, divisórias e linhas suaves |
| `--color-success` | `#1E5E46` | feedback positivo discreto |
| `--color-warning` | `#8B5E16` | alerta moderado |
| `--color-danger` | `#8A1F27` | alerta crítico controlado |

### 3.2 Regra de uso da cor

1. o acento vinho não deve dominar a tela;
2. a tinta escura é a cor de autoridade principal;
3. o neutro quente deve sustentar a identidade sem reduzir contraste;
4. o vermelho institucional não deve ser usado como “marketing urgente”.

---

## 4. Tipografia recomendada

## 4.1 Papéis tipográficos

Sugere-se separar papéis assim:

- **display/headings longos:** serif institucional;
- **body/UI:** sans estável e legível;
- **código/tabelas técnicas curtas:** mono discreta apenas quando necessário.

## 4.2 Stack sugerida

```css
--font-display: "Source Serif 4", Georgia, "Times New Roman", serif;
--font-body: "Inter", "Segoe UI", Arial, sans-serif;
--font-mono: "JetBrains Mono", "Fira Code", monospace;
```

## 4.3 Escala tipográfica sugerida

| Uso | Tamanho | Linha |
|---|---:|---:|
| Display XL | `3.5rem` | `1.05` |
| H1 | `2.75rem` | `1.1` |
| H2 | `2rem` | `1.15` |
| H3 | `1.5rem` | `1.2` |
| H4 | `1.25rem` | `1.3` |
| Body L | `1.125rem` | `1.75` |
| Body | `1rem` | `1.75` |
| Small | `0.9375rem` | `1.6` |
| Micro | `0.8125rem` | `1.5` |

### 4.4 Regra de composição

- headings podem usar a fonte display;
- texto corrido deve permanecer em fonte sans para leitura longa estável;
- parágrafos de artigos não devem ultrapassar largura confortável;
- evitar blocos densos demais sem intertítulos.

---

## 5. Layout, grid e espaçamento

## 5.1 Containers

| Container | Largura máxima |
|---|---:|
| `container-reading` | `72ch` |
| `container-content` | `1120px` |
| `container-wide` | `1280px` |

## 5.2 Escala de espaçamento

```css
--space-1: 0.25rem;
--space-2: 0.5rem;
--space-3: 0.75rem;
--space-4: 1rem;
--space-5: 1.5rem;
--space-6: 2rem;
--space-7: 3rem;
--space-8: 4rem;
--space-9: 5rem;
```

### 5.3 Regra estrutural

- artigos devem privilegiar a largura de leitura;
- páginas de listagem podem usar grid 2 ou 3 colunas em desktop;
- em mobile, reduzir complexidade em vez de comprimir demais.

---

## 6. Bordas, raio e sombra

```css
--radius-sm: 8px;
--radius-md: 14px;
--radius-lg: 22px;

--shadow-soft: 0 8px 24px rgba(0, 0, 0, 0.06);
--shadow-hover: 0 14px 34px rgba(0, 0, 0, 0.10);
```

Regra:

- borda e raio devem sugerir acabamento, não “app lúdico”;
- sombras mínimas e funcionais;
- evitar esqueumorfismo e excesso de profundidade.

---

## 7. Componentes mínimos obrigatórios

## 7.1 Header

Deve conter:

- marca;
- navegação principal curta;
- CTA contido;
- comportamento sticky apenas se leve e não intrusivo.

## 7.2 Footer

Deve conter:

- marca textual;
- rotas institucionais essenciais;
- contato;
- aviso de natureza informativa do conteúdo, quando oportuno;
- política de privacidade e páginas correlatas, quando disponíveis.

## 7.3 Card de área

O card de área deve ter, no mínimo:

- nome canônico da área;
- resumo de 2 a 3 linhas;
- link interno;
- mesma estrutura para todas as 7 áreas.

## 7.4 Card de post

Deve exibir:

- categoria;
- título;
- descrição curta;
- data;
- tempo de leitura, se houver;
- autor, quando relevante.

## 7.5 Shell de artigo

O template de artigo deve comportar:

- breadcrumb;
- categoria;
- H1;
- descrição;
- metadata;
- bloco de autor;
- conteúdo;
- bloco de relação com área;
- CTA contido;
- artigos relacionados.

## 7.6 CTA block

CTA deve aparecer como bloco de enquadramento, não de pressão.

Estrutura recomendada:

- título curto;
- 1 parágrafo de enquadramento;
- botão primário sóbrio;
- opção secundária discreta.

## 7.7 Note / disclaimer box

Usar para:

- observações documentais;
- ressalvas de dependência do caso concreto;
- distinção entre informação geral e análise jurídica.

---

## 8. Interação e movimento

## 8.1 Regra geral

Movimento deve ser raro, funcional e leve.

Permitido:

- hover suave;
- foco visível;
- transição curta de cor e sombra.

Evitar:

- parallax;
- animação de entrada excessiva;
- scroll hijacking;
- microefeitos contínuos.

### 8.2 Tempos recomendados

```css
--motion-fast: 120ms;
--motion-base: 180ms;
--motion-slow: 240ms;
```

---

## 9. Acessibilidade mínima obrigatória

1. contraste AA para textos e elementos interativos;
2. foco visível em teclado;
3. hierarquia de heading correta;
4. tamanho de toque confortável;
5. estados de erro legíveis;
6. imagens com `alt`;
7. tabelas só quando realmente úteis;
8. nenhum conteúdo relevante dependente apenas de cor.

## 11. Critérios de aceitação visual

Uma implementação visual só deve ser aprovada quando:

1. a home, `/sobre`, `/areas/`, páginas de área e posts parecem parte do mesmo sistema;
2. o conteúdo é confortável para leitura longa;
3. a navegação é clara em mobile;
4. não há explosão de cores, badges ou variações arbitrárias;
5. o CTA parece institucional, não mercantilizado;
6. o sistema suporta evolução sem refatoração total.

---

## 12. Cláusula final

O design da PAVIE deve operar como infraestrutura de confiança. Quando houver dúvida entre “mais efeito” e “mais legibilidade”, prevalece a segunda opção.
