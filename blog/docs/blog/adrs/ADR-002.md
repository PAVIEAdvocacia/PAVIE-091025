# ADR-002 — Decap CMS (git) + GitHub OAuth via Cloudflare Pages Functions
**Status**: Aprovado  
**Data**: 2025-11-03  
**R/A**: R=CMS-Admin, A=DevOps/Cloudflare  

## Contexto
Necessidade de CMS git-based sem lock-in, com login GitHub no próprio domínio.

## Decisão
Usar **Decap CMS** com backend **GitHub** e fluxo OAuth servido por **Pages Functions** (`/api/auth` e `/api/callback`), com **Cloudflare Access** em `/blog/admin`.

## Alternativas
- Netlify Identity (fora do stack).  
- OAuth de terceiros (Workers externos).

## Consequências
+ Admin integrado, baixo custo, privacidade.  
− Manter função OAuth e segredos.
