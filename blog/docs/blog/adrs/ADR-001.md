# ADR-001 — Astro + Content Collections (Zod)
**Status**: Aprovado  
**Data**: 2025-11-03  
**R/A**: R=FE-Astro, A=PM  

## Contexto
Garantir tipagem e validação do conteúdo em tempo de build, com erros claros ao violar o schema.

## Decisão
Adotar **Astro** com **Content Collections (Zod)** como base do blog. Schemas obrigatórios por coleção.

## Alternativas
- Markdown solto sem schema (mais frágil).  
- Outros SSGs (Next/Eleventy/Hugo) — maior esforço de migração.

## Consequências
+ Tipagem e DX robustas.  
− Aprendizado inicial do API de Collections.
